# 管理面板 UI 重设计（反代核心·安全中心）

## Goal

参照设计稿（深色科技风「反代核心·安全中心」仪表盘）重做现有 Emby 反代管理面板的界面，
覆盖**桌面深色 / 桌面浅色 / 移动端**三套视图。在不破坏现有功能的前提下，
把当前的单列卡片堆叠布局升级为「左侧导航 + 顶部状态栏 + 分区内容」的仪表盘布局。

## Background / 现状

- 整个管理面板是内嵌在 `worker.js` 中的一段 HTML/CSS/JS 字符串（约 909–3044 行）。
- 现有布局：Apple 风格单列卡片堆叠；移动端 4 个底部 tab（节点 / 测速 / 数据 / 设置）。
- 主题：`:root` 默认浅色，`@media (prefers-color-scheme: dark)` 提供深色覆盖。
- 依赖：`sortablejs`、`chart.js`（均为 CDN 引入）。

## Decisions（已与用户确认）

1. **侧边栏**：只保留真实存在的功能项，不放未实现的占位页。
2. **默认主题**：跟随系统（`prefers-color-scheme`），保留手动切换能力。
3. **改动边界**：视觉重样式 + 布局重构；允许适度调整 JS（tab/导航切换、渲染函数），
   但**不改后端接口、数据结构、业务逻辑**。

## Requirements

### R1 整体布局（桌面）
- 左侧固定侧边栏：Logo + 产品名「反代核心·安全中心 / Emby 反向代理管理喵板」+ 导航项 + 折叠按钮 + 版本号。
- 顶部状态栏：运行状态、节点总数、今日流量、健康度、admin 用户区。
- 主内容区按导航分区显示，切换导航即切换分区。
- 侧边栏可折叠（收起侧栏按钮）。

### R2 导航项（仅现有功能）
- 概览（节点管理为主视图）
- 线路测速 & 动态 DNS
- 数据统计
- 系统设置
- 工具箱（现有 cURL 导入等工具，若已存在则保留）
- 危险操作区作为常驻底部条，不作为独立导航。

### R3 视觉风格
- 深色科技风为主视觉：深底、霓虹蓝主色、卡片描边/微光、状态徽章（在线/延迟/离线）。
- 节点卡片：节点名 + 状态徽章 + 地区 + 延迟 + 流量 + 迷你折线图 + 操作按钮。
- 浅色模式为对应的明亮版本，保持同一套布局与组件。

### R4 移动端
- 保留底部 tab 导航（节点 / 数据 / 设置 三页或现有四页），样式按设计稿移动端卡片重做。
- 顶部精简状态栏；节点列表为紧凑卡片。

### R5 兼容性
- 所有现有功能（节点增删改、测速、DNS 更新、备份恢复、部署覆盖等）行为不变。
- `chart.js` / `sortablejs` 继续可用。
- 现有 DOM id / 事件绑定如被结构调整影响，需同步更新对应 JS。

## Constraints

- 单文件 `worker.js`，HTML 以模板字符串嵌入；注意字符串转义（反引号、`${}`）。
- 不引入构建步骤；新增依赖须为 CDN 或纯内联。
- 改动后 worker 仍能正常返回页面且无 JS 报错。

## Acceptance Criteria

- [ ] 桌面端呈现左侧导航 + 顶部状态栏 + 分区内容布局，视觉接近设计稿深色风格。
- [ ] 主题跟随系统，且手动深/浅切换均正常，浅色为明亮版同布局。
- [ ] 侧边栏导航切换分区正常；侧边栏可折叠。
- [ ] 节点卡片含状态徽章/地区/延迟/流量/迷你图/操作，数据由现有接口正常填充。
- [ ] 移动端三套视图按设计稿重做，底部 tab 正常切换。
- [ ] 所有现有功能（增删节点、测速、DNS、备份、部署）实测无回归，无 JS 控制台报错。
- [ ] 危险操作区常驻且样式醒目。

## Notes

- 设计稿仅为视觉参考，节点示例数据（上海/香港节点等）以真实接口数据为准。
